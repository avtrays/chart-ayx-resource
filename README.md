# chart-ayx-resource

本仓库用于归档和发布多个独立的 HTML 页面。这些页面内容互不关联，不针对任何特定域名或网站，仅作为资源存放与展示之用。

## 项目简介

chart-ayx-resource 是一个轻量级的静态页面归档仓库。主要用途包括：

- 存放结构完整的 HTML 文件
- 提供基于 GitHub Pages 的页面访问能力
- 便于对历史页面版本进行管理与回溯

所有页面均为独立文件，无后端依赖，可直接通过浏览器打开。

## 目录说明

```
.
├── index.html          # 仓库入口页面（可选）
├── pages/              # 存放主要 HTML 页面
├── assets/             # 公共资源（样式、脚本、图片等）
├── archive/            # 历史版本或不再更新的页面
└── README.md           # 本文件
```

- `pages/`：当前活跃的页面归档。
- `assets/`：页面引用的公共资源文件。
- `archive/`：已归档的旧版本页面，不再更新但保留供查阅。

## 页面归档说明

- 每个 HTML 页面均为独立完整文件，可脱离本仓库单独运行。
- 页面内容不指向任何具体业务、网站或服务。
- 归档遵循“提交即归档”原则：每次 commit 对应一个或多个页面的版本快照。
- 页面命名建议采用语义化名称，便于识别内容类别。

## 维护说明

- 本仓库由维护者不定期更新。
- 欢迎提交 Pull Request 添加新的独立页面，但需确保内容不涉及具体域名或商业推广。
- 提交前请确认页面无外部敏感链接或依赖。
- 仓库不保证页面内容的长期可用性或兼容性。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/chart-ayx-resource.git
   ```
2. 直接在浏览器中打开任意 `.html` 文件。
3. 若启用 GitHub Pages，可通过仓库设置中的 Pages 功能获得在线访问地址。
